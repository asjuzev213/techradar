Technology Radar Techniques — Апрель 2025

Кольца радара (перевод):
  • Принять (Adopt): Рекомендуется к внедрению.
  • Опробовать (Trial): Стоит изучить и протестировать.
  • Оценить (Assess): Требует анализа потенциала.
  • Приостановить (Hold): Не рекомендуется к использованию.

Принять (Adopt)

Техника IT-продукты Описание и рекомендация
1. Data product thinking DataHub, Collibra, Atlan, Informatica Подход к управлению данными как продуктом. Рекомендуется для масштабирования AI-инициатив и управления жизненным циклом данных.
2. Fuzz testing в тексте не найдено, но есть варианты: AFL, libFuzzer, Jazzer Тестирование с использованием некорректных входных данных. Рекомендуется для повышения безопасности AI-генерируемого кода.
3. Software Bill of Materials (SBOM) Syft, Trivy, Snyk, FOSSA Генерация SBOM для отслеживания зависимостей. Стандартная практика для безопасности ПО и AI-систем.
4. Threat modeling Microsoft Threat Modeling Tool, OWASP Threat Dragon Анализ угроз для безопасной разработки. Критически важен для AI-приложений.

Опробовать (Trial)

Техника IT-продукты Описание и рекомендация
5. API request collection as API product artifact Postman, Bruno, Insomnia Сбор API-запросов для улучшения DX. Рекомендуется интегрировать в CI/CD.
6. Architecture advice process в тексте не найдено, но есть варианты: ADR Tools, Architectural Decision Records Децентрализованный подход к архитектурным решениям. Рекомендуется для снижения бюрократии.
7. GraphRAG Neo4j GraphRAG, Graphiti Улучшение ответов LLM через графы знаний. Эффективен для анализа legacy-кода.
8. Just-in-time privileged access management в тексте не найдено, но есть варианты: CyberArk, HashiCorp Vault Минимизация привилегий для повышения безопасности. Рекомендуется для критичных систем.
9. Model distillation DeepSeek R1, Hugging Face DistillBERT Сжатие моделей для edge-устройств. Рекомендуется для оптимизации затрат.
10. Prompt engineering в тексте не найдено, но есть варианты: PromptLayer, LangChain Оптимизация запросов к LLM. Требует осторожности при автоматизации.
11. Small language models Meta Llama 3.2, Microsoft Phi-4, Google PaliGemma 2 Компактные модели для задач с ограничениями. Рекомендуется для edge-вычислений.
12. Using GenAI to understand legacy codebases GitHub Copilot, Sourcegraph Cody Анализ legacy-кода с помощью GenAI. Ускоряет модернизацию систем.

Оценить (Assess)

Техника IT-продукты Описание и рекомендация
13. AI-friendly code design в тексте не найдено, но есть варианты: SonarQube, CodeClimate, DeepSource Дизайн кода для совместимости с AI-ассистентами. Следует придерживаться best practices.
14. AI-powered UI testing QA.tech, LambdaTest KaneAI, Browser Use Тестирование UI с использованием LLM. Рекомендуется для динамичных интерфейсов.
15. Competence envelope в тексте не найдено, но есть варианты: Chaos Engineering tools (Gremlin) Анализ устойчивости систем. Рекомендуется для критически важных приложений.
16. Structured output from LLMs Pydantic, Zod, JSON Schema Форматирование вывода LLM. Снижает риск галлюцинаций.

Приостановить (Hold)

Техника IT-продукты Описание и рекомендация
17. AI-accelerated shadow IT в тексте не найдено, но есть варианты: No-code AI платформы Риск неконтролируемого использования AI. Требуется строгий governance.
18. Complacency with AI-generated code в тексте не найдено, но есть варианты: GitClear, Snyk, SonarQube Избегайте неконтролируемого доверия к AI-коду. Требуется ручной review.

20. Replacing pair programming with AI GitHub Copilot AI не заменяет коллаборацию. Рекомендуется для вспомогательных задач.
21. Reverse ETL в тексте не найдено, но есть варианты: Apache Airflow, dbt Риск централизации логики. Рекомендуется избегать для транзакционных систем.
